# 飞书协作 Skill

## 目标

将现有专利 Agent 的同一套状态机暴露到飞书，不创建与网页结果不一致的第二套写作流程。

## 会话规则

1. 定时任务按计划中配置的知识库和创新档位自动启动并推进到候选 idea；候选 idea 和章节结论仍由用户确认。
2. 非计划时间内，用户发送“开始生成”可随时启动一次新的交互流程。
3. 用户手动开始后，先选择知识库，再选择低、中、高创新档位。
4. 候选 idea 必须逐个完整展示，再等待用户选择。
5. 每个章节必须完整展示；过长内容分多条连续发送，不得摘要、截断或用省略号代替。
6. 每章展示后只接受四类动作：接受、重写、按意见修改、结束并保存。
7. 用户接受后自动生成下一章，不要求额外发送“继续”。
8. 所有飞书操作复用应用运行 API，因此候选、选择、章节版本、修改意见和最终文件进入同一份历史记录。
9. 最终提供 Markdown、Word、相似专利分析和完整历史链接；没有可访问的应用地址时，明确提示到应用历史记录下载。
10. 群聊仅响应用户 @机器人后的消息；私聊直接响应。用户发送“查看会话ID”时，返回群 Chat ID 和用户 Open ID，便于配置定时目标。

## 定时规则

1. 周期计划只允许用户通过每天、每周或每月及发送时间进行可视化配置，系统负责生成并校验底层调度规则。
2. 每周计划记录用户选择的星期；每月计划记录用户选择的 1 至 31 日；发送时间使用用户所在设置时区。
3. 倒计时计划支持按分钟或小时设置，并转换为只执行一次的绝对时间。
4. 所有计划必须展示下一次实际执行时间；一次性计划执行后标记完成，周期计划继续计算下一次时间。
5. 到点后直接调用现有专利 Agent，按计划中的知识库和创新档位推进到候选 idea，不得仅发送提醒而不启动生成。

## 安全边界

- App Secret 仅保存于当前用户运行时配置，不写入 Skill、代码或历史记录。
- 定时规则由用户创建、编辑、启停和删除，不在代码中写死频率。
- 机器人不会在用户未确认候选或章节时替用户作出决定。
